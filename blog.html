Margins, Borders and Padding

These three properties seem to be used a lot in Web Development and while being super useful by providing space between elements, can also cause a number of unexpected issues if not used in the right way as I found out the hard way. I understood these properties best when viewed graphically:

Together they make up what's called the "Box-Model". Each element has a box around it that consists of these properties and the content itself in the middle as displayed above.

Content is the actual stuff inside the element like images and text.

Padding is space added between the content and its border.

Border is displayed after the padding but is set to 0 by default.

Margin is the space outside the border.

Issues

Box-sizing

The biggest issue I had with this model was when it came to the box dimensions itself. I would set the width of something to fit neatly amongst other content only to have it bounce around the page unexpectedly.

After a bit of investigation I found out about a property called box-sizing. By default, box-sizing is set to content-box and the width and height of an element is calculated like this:

width + padding + border = actual width of an element
height + padding + border = actual height of an element

Meaning when you set the width of an element, the element often appears bigger than you have set (because the element's border and padding are added to the element's specified width/height).

To fix this and avoid having to compute the "actual" width/height of an element, we can set box-sizing:border-box. This allows us to include the padding and border in an element's total width and height. I personally set every element to border-box using the * selector for "all" so that I never have to worry about this issue again.
