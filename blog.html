Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'. (But not those ones).

Put simply, control flow is the order in which we perform or execute a set of tasks. loops are a way of repeating tasks until a certain criteria is met. Take washing the dishes for example; The normal flow of control would be to:
  1. Fill sink with hot water
  2. Add soap
  3. Take a dirty item
  4. Soak in water
  5. Scrub until clean
  6. Leave to dry
Now this flow may work for most dishes but what if we had say a cast iron skillet that shouldn't be washed with soap? We would have to divert the flow for this dish around step 2 and go straight to step 3 there by changing the flow depending on dish type. Step 5 is also an example of looping where you repeat the task of scrubbing until the criteria of the dish being clean is met. These are just a couple of examples of looping and control flow of which there could be many more in this scenario. What if the water is too hot? What if a dish breaks? What if the humidity is so high our dishes aren't drying? How would these effect the control flow?

Describe what the DOM is and an example of how you might interact with it.

The way I see the DOM is a hierarchy of objects or nodes that provide the structure to our page. The hierarchy can often been seen in the way we indent our HTML documents and is comparable to a family tree is the sense that each node can have children nodes and those children can have their own nodes etc. These nodes also have their own attributes similarly to how people in your family have attributes like their name, hair colour, height, etc. We can use these attribute fields to change how our nodes look and behave using CSS and Javascript.

Explain the difference between accessing data from arrays and objects.

Data in an array is typically accessed using an index value. The index value is the items position within the array. For example, cars[0] would give you to first car in the array.
Data in an object is typically accessed using dot(".") notation. For example, car.colour would give you the colour of the car.

Margins, Borders and Padding

These three properties seem to be used a lot in Web Development and while being super useful by providing space between elements, can also cause a number of unexpected issues if not used in the right way as I found out the hard way. I understood these properties best when viewed graphically:

Together they make up what's called the "Box-Model". Each element has a box around it that consists of these properties and the content itself in the middle as displayed above.

Content is the actual stuff inside the element like images and text.

Padding is space added between the content and its border.

Border is displayed after the padding but is set to 0 by default.

Margin is the space outside the border.

Issues

Box-sizing

The biggest issue I had with this model was when it came to the box dimensions itself. I would set the width of something to fit neatly amongst other content only to have it bounce around the page unexpectedly.

After a bit of investigation I found out about a property called box-sizing. By default, box-sizing is set to content-box and the width and height of an element is calculated like this:

width + padding + border = actual width of an element
height + padding + border = actual height of an element

Meaning when you set the width of an element, the element often appears bigger than you have set (because the element's border and padding are added to the element's specified width/height).

To fix this and avoid having to compute the "actual" width/height of an element, we can set box-sizing:border-box. This allows us to include the padding and border in an element's total width and height. I personally set every element to border-box using the * selector for "all" so that I never have to worry about this issue again.
